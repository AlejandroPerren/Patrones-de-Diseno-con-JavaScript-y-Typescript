Patrón Decorador

El Patrón Decorador es un patrón de diseño estructural que permite añadir funcionalidad adicional a un objeto de manera dinámica, sin modificar su estructura original. Este patrón es muy útil cuando queremos extender el comportamiento de clases de forma flexible.

¿Para qué sirve?

El patrón Decorador se utiliza cuando:

  • Queremos añadir responsabilidades o comportamientos a objetos individuales sin afectar a otros objetos de la misma clase.
  • No queremos modificar directamente la clase base o subclases existentes.
  • Necesitamos múltiples combinaciones de funcionalidades, que pueden extenderse dinámicamente.

¿Cómo funciona?

El patrón Decorador se basa en los siguientes conceptos:

  • Una clase componente base que define la interfaz común.
  • Una clase concreta que implementa la funcionalidad principal.
  • Una clase decoradora que extiende el comportamiento de la clase base de forma dinámica.
